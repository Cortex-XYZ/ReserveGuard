# ReserveGuard v0.1.0-alpha.1

ReserveGuard `v0.1.0-alpha.1` is the first public field-testing baseline for Monad developers.

This alpha is intentionally small. The goal is to get the current V1 primitives into real experiments and learn which reserve-aware patterns should become future library features.

## Included

- MIP-4 precompile interface at `0x1001`
- `ReserveGuard.dipped()`
- `ReserveGuard.assertHealthy()`
- `ReserveGuard.checkpoint()`
- `ReserveAware`
- `ReserveProtected`
- `ReserveViolation()`
- developer-facing examples
- Monad testnet experiment contracts
- EIP-7702 delegated drain/restore workflow
- optional ReserveGuard Lab browser app
- CI for Foundry contracts and the lab app

## Not Included

- named checkpoints
- checkpoint events
- non-reverting observe helper
- automatic recovery policies
- TypeScript SDK
- production monitoring or analytics

## Field Testing

Use this alpha to test real Monad flows and report what you observe:

- EIP-7702 delegated EOAs
- agent wallets
- routers
- vaults
- batch executors
- paymaster or sponsorship flows
- settlement paths that move native MON mid-transaction

Report results using `docs/field-testing.md` or the GitHub issue templates.

## Verification

The alpha baseline passes:

```bash
forge fmt --check
forge build
forge test
cd app && npm run typecheck
cd app && npm run build
```

Live Monad testnet observations remain the authority for real reserve-state semantics.
